================================================================
NGÀNH CNTT LÀ GÌ?
================================================================
Công nghệ Thông tin (CNTT / IT - Information Technology) là ngành học
nghiên cứu cách sử dụng máy tính và phần mềm để xử lý, lưu trữ,
truyền tải và bảo mật thông tin.

Ngành IT bao gồm nhiều chuyên ngành con:
1. Lập trình / Phát triển phần mềm (Software Development)
2. Khoa học dữ liệu & AI (Data Science / Machine Learning)
3. An toàn thông tin / Bảo mật (Cybersecurity)
4. Quản trị mạng & hệ thống (Network / System Admin)
5. Điện toán đám mây (Cloud Computing)
6. Phát triển Web (Web Development - Frontend & Backend)
7. Phát triển ứng dụng di động (Mobile Development)
8. DevOps / SRE
9. Kiểm thử phần mềm (QA / Testing)
10. UI/UX Design (kỹ năng mềm nhưng cần thiết cho IT)

LỘ TRÌNH HỌC NĂM NHẤT (Khuyến nghị)
================================================================

--- HỌC KỲ 1 ---
Mục tiêu: Xây dựng nền tảng tư duy lập trình

Môn học thường gặp:
- Nhập môn lập trình (Python hoặc C/C++)
- Toán rời rạc (Discrete Mathematics)
- Đại số tuyến tính (Linear Algebra)
- Tiếng Anh chuyên ngành

Kỹ năng tự học song song:
- Học Python cơ bản (miễn phí trên CS50P, W3Schools, freeCodeCamp)
- Làm quen với Git & GitHub (lưu code, portfolio)
- Cài đặt VS Code, làm quen môi trường lập trình
- Làm 5-10 bài LeetCode Easy để rèn tư duy thuật toán

--- HỌC KỲ 2 ---
Mục tiêu: Mở rộng ngôn ngữ, hiểu cấu trúc dữ liệu

Môn học thường gặp:
- Cấu trúc dữ liệu & Giải thuật (DSA)
- Lập trình hướng đối tượng (OOP)
- Cơ sở dữ liệu (SQL cơ bản)
- Vật lý / Xác suất thống kê

Kỹ năng tự học song song:
- Học HTML/CSS/JavaScript cơ bản (nếu hướng Web)
- Thực hành SQL trên SQLiteOnline hoặc MySQL
- Tạo dự án nhỏ đầu tiên (calculator, todo app, quiz app)
- Tham gia cộng đồng: Stack Overflow, Vietnam IT Community

CÁC NGÔN NGỮ LẬP TRÌNH PHỔ BIẾN & KHI NÀO DÙNG
================================================================

Python:
- Dễ học, cú pháp gần với tiếng Anh
- Dùng cho: AI/ML, Data Science, Backend, tự động hóa
- Phù hợp: Người mới bắt đầu, SV năm nhất
- Học ở đâu: CS50P (Harvard), Python.org, freeCodeCamp

JavaScript:
- Ngôn ngữ duy nhất chạy trực tiếp trên trình duyệt
- Dùng cho: Web Frontend (React, Vue), Backend (Node.js)
- Framework phổ biến: React, Next.js, Vue.js, Express.js

Java:
- Mạnh về OOP, dùng nhiều trong doanh nghiệp
- Dùng cho: Android App, Backend doanh nghiệp (Spring Boot)
- Thường được dạy trong các trường đại học Việt Nam

C/C++:
- Gần với phần cứng, tốc độ cao
- Dùng cho: Lập trình nhúng, game engine, hệ thống
- Khó học nhưng giúp hiểu sâu về bộ nhớ, con trỏ

SQL:
- Không phải ngôn ngữ lập trình nhưng BẮT BUỘC phải biết
- Dùng để truy vấn và quản lý cơ sở dữ liệu (MySQL, PostgreSQL)

C#:
- Của Microsoft, dùng cho game (Unity), app Windows, backend

Go / Rust:
- Ngôn ngữ hiện đại, hiệu suất cao
- Nên học sau khi đã vững 1-2 ngôn ngữ khác

CÔNG CỤ LẬP TRÌNH CẦN BIẾT
================================================================

Môi trường phát triển (IDE / Editor):
- VS Code: Phổ biến nhất, miễn phí, nhiều extension
- PyCharm: Chuyên cho Python
- IntelliJ IDEA: Chuyên cho Java
- Sublime Text: Nhẹ, nhanh
- Cursor: IDE tích hợp AI (xu hướng mới 2024-2025)

Quản lý phiên bản (Version Control):
- Git: Bắt buộc phải học! Giúp lưu lịch sử code, cộng tác nhóm
- GitHub: Lưu trữ code online, tạo portfolio, tham gia open source
- Lệnh Git cơ bản: git init, git add, git commit, git push, git pull

Terminal / Command Line:
- Windows: PowerShell hoặc Command Prompt
- Học lệnh cơ bản: cd, ls/dir, mkdir, cp, mv, rm
- Linux/Mac: Bash Terminal (nên học nếu muốn làm DevOps/Cloud)

Quản lý thư viện:
- pip (Python): pip install ten_thu_vien
- npm (JavaScript/Node.js): npm install ten_thu_vien
- Maven/Gradle (Java)

Cơ sở dữ liệu:
- MySQL Workbench: Giao diện để làm việc với MySQL
- DB Browser for SQLite: Nhẹ, phù hợp học tập
- MongoDB Compass: Cho cơ sở dữ liệu NoSQL

LỜI KHUYÊN THỰC TIỄN CHO SV NĂM NHẤT
================================================================

1. ĐỪNG HỌC VẸT - HÃY THỰC HÀNH
   Đọc 100 trang sách không bằng tự gõ 100 dòng code.
   Học xong lý thuyết → làm ngay 1 bài tập liên quan.

2. XÂY DỰNG PORTFOLIO SỚM
   Tạo tài khoản GitHub ngay từ năm nhất.
   Mỗi dự án nhỏ đều nên đẩy lên GitHub (dù chỉ là todo app).
   Nhà tuyển dụng rất chú trọng portfolio thực tế.

3. HỌC TIẾNG ANH SONG SONG
   80% tài liệu IT là tiếng Anh.
   Không cần nói như người bản xứ, nhưng phải đọc-hiểu tốt.
   Luyện đọc documentation, Stack Overflow mỗi ngày.

4. THAM GIA CỘNG ĐỒNG
   - Câu lạc bộ IT của trường
   - Hackathon (thi lập trình 24-48h, rất có giá trị kinh nghiệm)
   - Group Facebook IT VN: "Lập trình không khó", "Dân IT"
   - GitHub open source contributions

5. ĐỪNG SO SÁNH MÌNH VỚI NGƯỜI KHÁC QUÁ MỨC
   Mỗi người có lộ trình riêng. Tập trung vào việc mình giỏi hơn
   BẢN THÂN HÔM QUA, không phải giỏi hơn người khác.

6. INTERNSHIP / THỰC TẬP SỚM
   Cố gắng có thực tập từ năm 2-3.
   1 tháng thực tập thực tế = 6 tháng tự học ở nhà.
   Tìm trên TopCV, ITviec, LinkedIn, Facebook Groups.

7. KIÊN NHẪN VỚI BUG
   Bug (lỗi) là bạn đồng hành của lập trình viên.
   Khi gặp bug: đọc thông báo lỗi → Google → Stack Overflow.
   Mỗi lần fix bug = học thêm được một điều mới.

8. QUẢN LÝ THỜI GIAN
   IT rất rộng, không ai học hết được.
   Chọn 1-2 hướng và đào sâu thay vì học lan man.
   Dùng kỹ thuật Pomodoro (25 phút học - 5 phút nghỉ).

CÁC THUẬT NGỮ IT CƠ BẢN
================================================================

API (Application Programming Interface):
Giao diện để các ứng dụng giao tiếp với nhau. Ví dụ: app thời tiết
gọi API để lấy dữ liệu từ máy chủ thời tiết.

Framework:
Bộ công cụ/thư viện được dựng sẵn để xây dựng phần mềm nhanh hơn.
Ví dụ: React (Frontend), Django/Flask (Python Backend), Spring (Java).

Library (Thư viện):
Tập hợp code đã viết sẵn để dùng lại. Nhỏ hơn Framework.
Ví dụ: NumPy (tính toán số học Python), requests (gọi API).

Bug:
Lỗi trong code khiến chương trình không hoạt động đúng.

Debug:
Quá trình tìm và sửa bug.

Algorithm (Thuật toán):
Chuỗi các bước logic để giải quyết một vấn đề.

Data Structure (Cấu trúc dữ liệu):
Cách tổ chức, lưu trữ dữ liệu. Ví dụ: Array, List, Stack, Queue, Tree.

OOP (Object-Oriented Programming - Lập trình hướng đối tượng):
Phong cách lập trình mô hình hóa thực thể thành "đối tượng" có
thuộc tính và phương thức. Ví dụ: Class SinhVien có thuộc tính tên,
tuổi và phương thức hocBai().

Database (Cơ sở dữ liệu):
Nơi lưu trữ dữ liệu có tổ chức. SQL (MySQL, PostgreSQL) và NoSQL (MongoDB).

Frontend:
Phần người dùng nhìn thấy và tương tác trực tiếp (giao diện web/app).

Backend:
Phần "phía sau" - logic xử lý, cơ sở dữ liệu, máy chủ.

Cloud:
Điện toán đám mây - thuê máy chủ/dịch vụ qua internet thay vì
mua phần cứng. AWS, Google Cloud, Azure là 3 nhà cung cấp lớn nhất.

Docker:
Công nghệ "đóng gói" ứng dụng vào container để chạy nhất quán
trên mọi môi trường.

Git:
Hệ thống quản lý phiên bản code - theo dõi mọi thay đổi trong code.

Open Source:
Phần mềm mã nguồn mở - code được công khai để mọi người xem, sửa,
đóng góp miễn phí. Linux, Python, React đều là open source.

HỎI & ĐÁP THƯỜNG GẶP
================================================================

Q: Năm nhất nên học ngôn ngữ gì đầu tiên?
A: Python! Cú pháp đơn giản, dễ đọc, ứng dụng rộng (AI, web, data).
   Sau khi vững Python thì học thêm JavaScript hoặc Java tùy hướng.

Q: Không giỏi Toán có học IT được không?
A: Có! Phần lớn công việc IT thực tế (web, mobile, backend) không cần
   Toán cao cấp. Chỉ AI/ML/Game/Graphics mới cần Toán nhiều.
   Toán tốt thì thuận lợi hơn, nhưng không giỏi Toán vẫn làm IT được.

Q: Bao lâu có thể xin việc được?
A: Nếu học tập nghiêm túc và có dự án thực tế:
   - Web/Mobile Dev: có thể xin intern sau 1-1.5 năm học đúng hướng
   - Tốt nghiệp đại học (4 năm) với portfolio tốt: xin job khá dễ
   Điều quan trọng nhất là CÓ DỰ ÁN THỰC TẾ để show nhà tuyển dụng.

Q: Nên học online hay offline?
A: Kết hợp cả hai! Học lý thuyết online (CS50, YouTube, Udemy),
   học thực hành + networking với bạn bè offline tại câu lạc bộ/hackathon.

Q: Freelance có khó không?
A: Có thể bắt đầu từ năm 2-3. Các platform: Upwork, Fiverr, Toptal.
   Bắt đầu từ dự án nhỏ, giá thấp để tích lũy review.
   Cần kỹ năng giao tiếp khách hàng và quản lý thời gian tốt.
